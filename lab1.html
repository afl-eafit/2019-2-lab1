Programming Lab 1: Representing Images with Deterministic Finite Automata

TA: Alejandro Murillo G.

Introduction

Deterministic Finite Automata (DFA) enable the representation of multiple structures. One of those are images, as described by Culik and Valenta (1996). The goal of this programming lab is to represent images, given its pixels, using DFAs.

Assignment (80%)

  • Download the GitHub repository 2019-2-lab1.

  • (70%) Using the data types in the above repository define the functions:

     img2dfa :: ImagePixels -> FiniteAutomata state symbol

    that takes an image's pixels and returns a DFA that describes the image;

     createImage :: Int -> FiniteAutomata state symbol -> DynamicImage

    that takes the image's size scale factor n and the DFA that describes the image, and returns an image created using the DFA.

    You will need the library JuicyPixels to generate and save the images.

  • (10%)
    • Your lab will be tested using the -Wall, -Wmissing-local-signatures and -Wmissing-signatures options.
    • Annotate (in English) your source code using Haddock. Documentation should explain your solution for the assignment.
    • Improve your implementation with HLint.

  • The programming lab may either be solved on your own, or jointly with one other student taking the course.

  • You are to use GHC[i] ≥ 8.6.5.

Examples

Using the FiniteAutomata.hs module in the above GitHub repository, this is what a sample input-output looks like:

      GHCi> cb = img2dfa chessBoard
      GHCi> cb_img = createImage 7 cb
      GHCi> savePngImage "chessBoard.png" cb_img

Once the automaton is generated, and its image created, you could use the savePngImage function to save the PNG image named "chessBoard" (a 128 x 128 white and black image) to the current working directory.
