Programming Lab 1: Converting Non-Deterministic Finite Automata to Deterministic Finite Automata

TA: Alejandro Murillo G.

Introduction

While non-deterministic finite automata and deterministic finite automata describe languages differently, both actually represent the same type of languages, the "regular languages" [Hopcroft, Motwani and Ullman 2007]. The goal of this programming lab is to convert NFAs to DFAs to show that both of them are equally powerful.

Assignment (80%)

  • Download the GitHub repository 2019-1-lab1.

  • (70%) Using the data types in the above repository define a function:

     nfa2dfa :: FiniteAutomata state symbol ->
        FiniteAutomata (Set state) symbol

    that takes a non-deterministic finite automaton and returns a deterministic finite automaton that defines the same language. The obtained DFA should only contain reachable states.

  • (10%)
    • Your lab will be tested using the -Wall, -Wmissing-local-signatures and -Wmissing-signatures options.
    • Annotate (in English) your source code using Haddock. Documentation should explain your solution for the assignment.
    • Improve your implementation with HLint.

  • The programming lab may either be solved on your own, or jointly with one other student taking the course.

  • You are to use GHC[i] ≥ 8.6.3.
